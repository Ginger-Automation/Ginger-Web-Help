Ginger on Website Overview Contents Description Pre Requisites How to use the Ginger Website Login Jobs List Create Job Run Job Job History Job Console ‎ Description The Ginger Website is created for allowing Ginger users to configure and trigger Ginger Executions from any machine without Installing Ginger. Ginger Website offers: Create a Job to create and schedule Ginger Run Set Execution   Pre Requisites Prerequisites are: Ginger Solution which: Configured on Ginger SVN/GIT Data Source Includes Run Set/s ready to be executed Jenkins Server version 1.606 or above with Ginger Plug 1.5 or above Installed Jenkins Slave installed on the machine hosting Ginger(Beta 2.5.9 or above) installation (machine also need to be able to access SVN server in which the Ginger Solution exist)   How to use the Ginger Website Login Solution Login Enter the Solution Source Control details. Jenkins Login Enter the Jenkins Login details below: Parameter Name Explanation URL Jenkins URL User Jenkins User Token Jenkins API Token Available in Jenkins User Configuration. Refer How to Generate API Token How to Generate API Token Login to Jenkins. ‎ Go to Configuration. ‎ Click on Show API Token. ‎ Copy the API Token ‎ ‎ If Jenkins has a valid ginger plug-in installed, Jobs List Page will be loaded.   Jobs List Job List Page act as a Home Page for Ginger Website. It lists all the jobs configured in the current Jenkins for the logged-in solution. It also displays the last execution details for each job.   Navigations from Jobs List You can perform the below operations from the Jobs List page: Create Job: To Create a new Job, click Job->New Job from Page Header. It will take you to Create Job Page. Quick Run Job: You can trigger the job directly from the Jobs List page by clicking Quick Run Button. It will execute the job with the last execution parameters.   Execution History: You can navigate to Job Execution History by clicking the Job Name/No of Executions link as shown below. Last Execution Job Console: You can navigate to the Last Job Execution Console by clicking on the Last execution status link from the Job List Table, as shown below. Job Actions: You can perform various actions for a job by clicking on the Actions button and select the operation you want to perform from below. Job Configuration: This allows you to edit the job configuration. Enable/Disable Job: This allows to enable or disable the Job. Delete Job: Delete the current Job. Run Job: Show the Run configuration and you can trigger the execution from the homepage itself Create Job Create Job Page allows you to create a new job to trigger/schedule execution for a particular run set.
