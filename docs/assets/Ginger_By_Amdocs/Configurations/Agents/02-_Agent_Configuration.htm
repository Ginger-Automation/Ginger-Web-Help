11.1 Run Agent in Hidden mode

 

This functionality is available only for the Chrome and Firefox Agents.

When the Agent is run (either from Automate tab or the Runset tab), it is not visible. Only its console is

visible.

To do this, select the required Agent. Click on the Driver Configuration

The parameter ‘HeadlessBrowserMode’ has to be made as ‘true’. By default, it is marked as false.

This mode is used for performing the execution faster.

 

 

 

11.2 Run Agent in Minimized mode

 

This functionality is available only for the Chrome and Firefox Agents.

When the Agent is run (either from Automate tab or the Runset tab), it is not visible. Only its console is

visible.

In the Agent’s Driver Configuration, the parameter ‘BrowserMinimized’ has to be made as ‘true’. By

default it is marked as false.

This mode is used when the user wants to work on some other window while the Ginger execution is

running.

How to map an Agent to a Business Flow

 

To use this Agent for the execution, we have to map the Agent to the Business Flows.

When a solution is created, the default agent is automatically mapped to the default Business Flow.

New Agent can be mapped in the following way:-

 

1) Map the agent to the Business Flow

Go to the Business Flows tab and double-click on the required Business Flow.

Click on the ‘Change Agent Mapping’ button as shown below.

 

2) The below window will be displayed where only those agents will be displayed whose platform type is the same as the target application platform type. Select the desired agent and click on the Map button to map the agent.
