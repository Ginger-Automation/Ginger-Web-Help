Description

         The analyzer will help you to find problems and issues within your Solution\Business Flow\Run Set.

         Problems that will prevent your flow from running correctly or getting failed.

         This problem can be defined as the wrong implementation.

 

Pre-Requisites

Minimum Ginger version 2.4

 

How to Run the Analyzer

Go to the Business Flow tab and double-click on the desired Business Flow to go to the Automation area.

When a Business Flow is Running, Ginger analyzes your Business Flow trying to find implementation Errors that will prevent your flow to run correctly.

When the user ON toggle button of Auto Analyse Business Flow before Flow Execution then analyzer will run before business flow and if the toggle button is OFF then analyzer will not run before business flow Same goes also for Run on “Run set” tab.

 

 

You can run the analyzer also by Clicking on the analyzer button both on a solution and Business flow, if you run it on the solution level Analyzer will include all business flows within this solution.

      Once your solution/ business flows are open, click on the Analyzer button.

 

The Analyzer Report

         Once the Analyzing processes are done, you can get the report by clicking on the ‘Analyse business Flow to Detect Integrity Issue’ from Analyzer Options.

        This report will list the issues that we have on our solution/Business flow/Activity/Action.

List of issues:

In the grid, you will be able to see the list of issues that were found sorted by severity Critical - > Low

If Critical/High issues are found this will prevent the Run to run correctly and your Flow will fail.

All issues that can be Auto fixed will be already selected in the grid (As shown in the pic).

Statistics:

Here u can find the total issues that were found, the number of high and critical and the amount of can be Auto fixed issues.

Issue description and detailed information:

Here u can find all the needed information regarding the founded issue:

Item Type – Solution/Business Flow/Activity/Action

Item Name – The name of the Solution/ Business Flow/Activity/Action that the issue was found on.

Item Parent – This will display the Parent of the Item. E.g., if the issue was found on Activity, then the parent will be the name of the Business Flow of that Activity.

Issue – what issue was found

Issue Details – Detailed description.

Issue Impact - how this issue will impact your Solution/BF/Activity/Action.

How To Fix

Can Auto Fix – If the issue can be Auto Fix then by clicking on Fix Selected button it will be fixed automatically.
